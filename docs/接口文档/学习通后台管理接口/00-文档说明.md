### 2.1 鉴权与权限

1. 除 `POST /api/admin/v1/auth/login` 与 `POST /api/admin/v1/auth/refresh` 外，其他接口均需要传入 `Authorization: Bearer <token>`。
2. 代码中通过 `@RequirePermission(...)` 控制菜单与操作权限，本文会在每个模块下标注对应权限点。
3. 登录与刷新接口支持可选请求头 `User-Agent`，用于记录会话环境。

### 2.2 通用返回结构

#### 2.2.1 通用成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "uuid",
  "timestamp": 1715400000000
}
```

#### 2.2.2 分页响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "total": 100,
    "list": []
  },
  "requestId": "uuid",
  "timestamp": 1715400000000
}
```

#### 2.2.3 简单布尔响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "success": true
  },
  "requestId": "uuid",
  "timestamp": 1715400000000
}
```

### 2.3 通用查询约定

| key | value | 说明 | 是否必填 | 备注 |
| --- | ----- | ---- | -------- | ---- |
| pageNum | int | 页码 | N | 默认 `1` |
| pageSize | int | 每页条数 | N | 默认 `10` |
| keyword | string | 关键字 | N | 不同模块用于用户名、标题、名称等模糊匹配 |
