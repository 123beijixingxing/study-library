# 学习通后台前后端对接清单

## 1. 文档说明

- 项目名称：学习通后台管理系统
- 对应前端工程：`study-library-admin`
- 对应接口文档：`docs/需求设计文档/学习通接口清单.md`

## 5. mock 数据层说明

- 当前 mock 数据主入口：`study-library-admin/src/mock/db.ts`
- mock 数据存储位置：浏览器 `localStorage`
- 存储键名：`study-library-admin-mock-db`
- 支持本地持久化的业务数据：用户、权限、首页配置、课程、课程详情、园地内容、举报、会员、套餐、成长规则、消息、客服工单、系统配置、题库、练习试卷、统计趋势、日志
- 当前 mock 行为已经支持基础增删改后保留，可用于产品演示和前端联调前自测

## 5.1 adapter 分层说明

- 当前前端已增加 adapter 层：`study-library-admin/src/api/adapters/admin.ts`
- 响应包装工具：`study-library-admin/src/api/utils/adapter.ts`
- 作用：统一将正式接口字段适配成页面使用字段，例如 `userId -> id`、`configId -> id`、`bankId -> id`
- 建议后端联调时尽量保持接口文档字段不变，由 adapter 层完成前端字段映射

## 6. 字段对接建议

当前前端页面内部为了开发便利，部分展示层字段仍采用较直接的命名，例如：

- 用户页使用 `id`，正式接口可返回 `userId` 后在前端适配为表格行 `id`
- 题库页当前使用 `packageId`、`paperId`、`reportId` 等正式风格字段，已更接近接口文档
- 若后端严格按接口文档字段返回，建议在 API 模块中增加一层 `adapter`，不要把后端字段改名逻辑分散写在页面组件内

推荐适配层位置：

- `study-library-admin/src/api/modules/*.ts`
- 或新增 `study-library-admin/src/api/adapters/*.ts`

## 7. 后端联调待补事项

当前前端已预留但后续联调仍需明确：

1. 用户详情接口中的 `studyRecordList`、`favoriteList` 明细字段结构
2. 课程详情接口中的章节内容、排序、富文本/视频字段结构
3. 举报处理接口中的 `handleResult` 枚举值和是否需要审核备注必填
4. 会员套餐中的 `benefitList` 最终是数组、富文本还是对象结构
5. 成长规则中的触发类型、每日上限、周期限制是否需要更多维度
6. 练习试卷是否需要返回成绩分布、提交人数、关联题库明细
7. 日志查询是否需要分页、时间范围、导出接口

## 8. 推荐联调顺序

3. 首页管理、课程管理、课程详情管理
4. 园地管理、举报审核
5. 会员、套餐、成长规则、消息、客服

## 9. 验收建议

- 联调前先使用 mock 跑通页面交互
- 每接通一个模块，先验证列表查询，再验证编辑保存，再验证状态切换
- 对带详情页的模块，优先保证主列表与详情页字段一致
- 对后端返回枚举值，统一收敛到 `study-library-admin/src/constants/enums.ts` 管理
