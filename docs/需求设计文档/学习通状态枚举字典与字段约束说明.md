## 2. 编写约定

| 项 | 说明 |
| --- | --- |
| 枚举值命名 | 统一使用小写英文或 `snake_case`，数据库、接口、前端常量保持一致 |
| 展示文案 | 页面展示可使用中文文案，但接口和数据库仅存英文枚举值 |
| 已明确约束 | 直接来源于需求文档、字段文档、接口文档或数据库设计文档 |
| 建议约束 | 当前文档补充的实现建议，用于减少开发歧义，后续可由产品或技术方案再确认 |
| 空值规则 | 可空字段使用 `NULL` 表示未知或未设置，不建议使用空字符串替代 |
| 默认值规则 | 无特别说明时，状态字段应给出明确默认值，数量字段默认 `0`，逻辑删除字段默认 `0` |

## 5. 字段约束说明

### 5.1 通用字段约束

| 字段 | 约束级别 | 约束说明 | 备注 |
| --- | --- | --- | --- |
| `id` | 已明确 | 使用 `bigint` 主键，正整数，自增或雪花 ID 均可 | 数据库方案优先统一一种生成策略 |
| `created_at` / `updated_at` | 已明确 | 使用 `datetime`，统一记录业务时间 | 所有主业务表都应具备更新时间 |
| `created_by` / `updated_by` | 建议约束 | 仅后台维护类表强制保留，前台业务表可选 | 便于审计 |
| `is_deleted` | 已明确 | 取值仅允许 `0`、`1`，默认 `0` | 逻辑删除表统一使用 |
| `sort_no` | 建议约束 | 非负整数，默认 `0` | 越小越靠前 |
| `status` 类字段 | 已明确 | 仅允许使用枚举字典中定义的值 | 禁止出现中文或大小写混用 |
| 数量类字段 | 建议约束 | `view_count`、`favorite_count`、`like_count` 等必须大于等于 `0` | 默认 `0` |
| 比例类字段 | 建议约束 | `progress_percent` 范围 `0.00`-`100.00` | 保留两位小数 |
| 时间区间字段 | 建议约束 | `effect_end_time >= effect_start_time` | 生效区间为空时表示长期有效 |

### 5.2 账号与认证字段约束

| 对象/字段 | 约束级别 | 约束说明 | 备注 |
| --- | --- | --- | --- |
| `sys_user.username` | 已明确 | 必填；长度 `1-30`；全局唯一；不能与登录密码明文相同 | 前后端双校验 |
| `sys_user.password_hash` | 已明确 | 数据库仅存哈希值；明文密码输入长度不超过 `18` | 建议服务端使用强哈希算法 |

| `sys_user.nickname` | 建议约束 | 长度建议 `1-50` | 可允许为空 |
| `sys_user.email` | 已明确 | 可为空；填写时必须符合邮箱格式；建议唯一 | 长度建议不超过 `100` |
