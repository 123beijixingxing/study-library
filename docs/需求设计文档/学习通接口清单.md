## 2. 接口设计约定

### 2.1 基础路径

| 端 | Base URL | 说明 |
| --- | --- | --- |
| 前台 App/H5 | `/api/app/v1` | 面向普通用户与会员用户 |
| 后台 Admin | `/api/admin/v1` | 面向管理员与超级管理员 |

### 2.2 鉴权方式

| 项 | 说明 |
| --- | --- |
| 用户登录态 | `Authorization: Bearer <app_token>` |
| 管理员登录态 | `Authorization: Bearer <admin_token>` |
| 免登录接口 | 登录、注册、找回密码、首页公开内容、公开搜索等 |
| 权限控制 | 后台接口需按角色与权限点控制；超级管理员具备最高权限但仍受不可删除自身等业务规则限制 |

### 2.3 通用请求参数

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNum` | number | 页码，默认 1 |
| `pageSize` | number | 每页条数，默认 10 或 20 |
| `keyword` | string | 关键字搜索 |
| `status` | string | 状态筛选 |
| `startDate` | string | 开始日期，格式 `YYYY-MM-DD` |
| `endDate` | string | 结束日期，格式 `YYYY-MM-DD` |
| `sortField` | string | 排序字段 |
| `sortOrder` | string | 排序方式：`asc` / `desc` |
| `tabType` | string | 标签页类型，如火热、分享、收藏 |
| `scene` | string | 场景标识，如首页搜索、学习页搜索、练习中心 |
| `dateType` | string | 统计时间维度，如 `today`、`7d`、`30d` |
| `messageType` | string | 消息类型，如系统、活动、服务 |
| `auditStatus` | string | 审核状态筛选 |

### 2.4 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "202604231030001234",
  "timestamp": 1776901800000
}
```

### 2.5 分页响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "total": 100,
    "list": []
  }
}
```

### 2.6 通用业务状态建议

| 字段 | 建议值 | 说明 |
| --- | --- | --- |
| 用户状态 | `enabled` / `disabled` | 启用、禁用 |
| 课程状态 | `draft` / `pending` / `published` / `offline` | 草稿、待审核、已上架、已下架 |
| 内容审核状态 | `pending` / `approved` / `rejected` | 待审核、已通过、已拒绝 |
| 会员状态 | `active` / `expiring` / `expired` / `cancelled` | 正常、即将到期、已过期、已取消 |
| 消息状态 | `unread` / `read` | 未读、已读 |
| 练习状态 | `not_started` / `in_progress` / `submitted` / `reviewed` | 未开始、进行中、已提交、已批改 |
| 工单状态 | `waiting` / `processing` / `resolved` / `closed` | 待处理、处理中、已解决、已关闭 |
| 举报状态 | `pending` / `processing` / `resolved` / `rejected` | 待处理、处理中、已处理、已驳回 |

## 3. 前台接口清单

### 3.1 认证与账号接口

| 接口名称 | Method | URL | 说明 | 关键请求参数 | 关键响应字段 |
| --- | --- | --- | --- | --- | --- |
| 用户登录 | `POST` | `/api/app/v1/auth/login` | 用户名密码登录 | `username`、`password`、`rememberMe` | `token`、`refreshToken`、`userInfo` |
| 用户退出登录 | `POST` | `/api/app/v1/auth/logout` | 注销当前登录态 | 无 | `success` |
| 用户注册 | `POST` | `/api/app/v1/auth/register` | 提交注册信息 | `username`、`password`、`confirmPassword`、`email`、`phone` | `userId`、`registerTime` |
| 发送找回密码验证码 | `POST` | `/api/app/v1/auth/password/code` | 按邮箱或手机号发送验证码 | `accountType`、`emailOrPhone` | `expireSeconds` |
| 校验找回密码验证码 | `POST` | `/api/app/v1/auth/password/verify` | 校验验证码是否正确 | `emailOrPhone`、`code` | `verifyToken` |
| 重置密码 | `POST` | `/api/app/v1/auth/password/reset` | 使用验证码校验令牌重置密码 | `verifyToken`、`newPassword`、`confirmPassword` | `success` |
| 获取当前用户信息 | `GET` | `/api/app/v1/auth/me` | 返回当前登录用户基础信息 | 无 | `userId`、`username`、`avatar`、`roleList` |

#### 3.1.1 登录请求参数建议

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | string | 是 | 用户名，长度不超过 30 |
| `password` | string | 是 | 密码，长度不超过 18 |
| `rememberMe` | boolean | 否 | 是否记住登录态 |

#### 3.1.2 登录响应字段建议

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `token` | string | 访问令牌 |
| `refreshToken` | string | 刷新令牌 |
| `userInfo.userId` | number | 用户 ID |
| `userInfo.username` | string | 用户名 |
| `userInfo.avatar` | string | 头像地址 |
| `userInfo.memberStatus` | string | 会员状态 |

## 7. 关键接口状态前置条件

以下约束用于补充“接口可在什么状态下调用”的实现边界。若前置状态不满足，建议统一返回业务错误码，并附带当前状态和值域说明。

### 7.1 通用校验建议

| 场景 | 建议处理 |
| --- | --- |
| 资源当前状态不允许执行该操作 | 返回 `409` 或业务码 `STATE_NOT_ALLOWED` |
| 目标状态值非法或不在可选值域内 | 返回 `400` 或业务码 `INVALID_STATUS` |
| 前台访问到未上架、未审核、已隐藏资源 | 优先返回 `404` 或 `403`，避免暴露资源内部状态 |
| 需要会员权限但用户非有效会员 | 返回 `403` 或业务码 `MEMBER_REQUIRED` |
| 同一操作重复执行但已处于目标状态 | 返回成功幂等响应，或返回明确提示“状态未变化” |

### 7.2 前台访问与提交类接口

| 接口 | 资源状态字段 | 允许前置状态 | 不允许状态 | 说明 |
| --- | --- | --- | --- | --- |
| `GET /api/app/v1/courses/{courseId}` | `edu_course.status` | `published` | `draft`、`pending`、`offline` | 前台仅可访问已上架课程 |
| `GET /api/app/v1/courses/{courseId}/chapters` | `edu_course.status`、`edu_course_chapter.status` | 课程为 `published`，章节为 `published` | 课程未上架或章节未发布/已下线 | 返回章节列表时应过滤非 `published` 章节 |
| `POST /api/app/v1/courses/{courseId}/evaluations` | `edu_course.status`、评价唯一性 | 课程为 `published`，且用户不存在同课程有效评价 | 课程未上架；同用户重复提交评价 | 建议与 `uk_course_user_eval(course_id, user_id)` 保持一致 |
| `GET /api/app/v1/garden/contents/{contentId}` | `garden_content.audit_status`、`garden_content.status` | `approved` 且 `visible` | `pending`、`rejected`、`hidden`、`offline` | 前台只读可见且审核通过内容 |
| `POST /api/app/v1/garden/contents/{contentId}/comments` | `garden_content.audit_status`、`garden_content.status` | `approved` 且 `visible` | 其他状态 | 不允许对隐藏、未审核内容评论 |
| `POST /api/app/v1/garden/contents/{contentId}/report` | `garden_content.audit_status`、`garden_content.status` | `approved` 且 `visible` | `offline`、物理不存在 | 举报成功后新建 `pending` 举报记录 |
| `GET /api/app/v1/practice/papers/{paperId}` | `practice_paper.status`、`practice_paper.member_only` | 试卷为 `published`；会员试卷要求用户会员状态为 `active` 或 `expiring` | `draft`、`disabled`；非会员访问会员试卷 | 建议在接口层统一校验会员权限 |
| `POST /api/app/v1/practice/papers/{paperId}/submit` | `user_practice_record.practice_status`、`practice_paper.status` | 试卷为 `published`，记录为 `not_started` 或 `in_progress` | `submitted`、`reviewed`、试卷 `draft/disabled` | 重复提交应幂等处理或明确拒绝 |
| `POST /api/app/v1/me/check-in` | `user_check_in_record.check_in_date` | 当日未签到 | 当日已签到 | 同一用户同一自然日只允许签到一次 |
| `PATCH /api/app/v1/me/messages/{messageId}/read` | `sys_message_receiver.read_status` | `unread` | `read` | 已读再次调用可按幂等成功处理 |
| `PATCH /api/app/v1/me/messages/read-all` | `sys_message_receiver.read_status` | 当前筛选范围存在 `unread` 消息 | 无未读消息 | 可返回成功且已读数量为 `0` |
| `POST /api/app/v1/me/customer-service/sessions/{sessionId}/messages` | `service_ticket.ticket_status` | `waiting`、`processing`、`resolved` | `closed` | 若当前为 `resolved`，用户追问后建议自动重开为 `processing` |

### 7.3 后台状态变更类接口

| 接口 | 资源状态字段 | 允许前置状态 | 目标状态/动作 | 说明 |
| --- | --- | --- | --- | --- |
| `PATCH /api/admin/v1/users/{userId}/status` | `sys_user.status` | `enabled`、`disabled` | `enabled <-> disabled` | 不建议允许管理员禁用自身账号 |
| `PATCH /api/admin/v1/courses/{courseId}/status` | `edu_course.status` | `published`、`offline` | `published <-> offline` | 若首版未启用审核流，可临时支持 `draft -> published`；启用审核流后建议由审核动作完成 `pending -> published` |
| `PATCH /api/admin/v1/course-evaluations/{evaluationId}/status` | `edu_course_evaluation.status` | `visible`、`hidden` | `visible -> hidden`、`hidden -> visible`、`visible/hidden -> deleted` | `deleted` 视为终态，不建议恢复 |
| `POST /api/admin/v1/contents/{contentId}/audit` | `garden_content.audit_status` | `pending` | `approved` 或 `rejected` | 非 `pending` 内容不应重复审核 |
| `POST /api/admin/v1/members/{userId}/grant` | `user_member_record.status` | 无记录、`expired`、`cancelled` | 开通为 `active` | 对 `active/expiring` 用户建议走续期或叠加策略，避免直接覆盖 |
| `POST /api/admin/v1/members/{userId}/cancel` | `user_member_record.status` | `active`、`expiring` | 变更为 `cancelled` | 已 `expired/cancelled` 不应重复取消 |
| `PATCH /api/admin/v1/questions/{questionId}/status` | `practice_question.status` | `draft`、`published`、`disabled` | `draft -> published`、`published -> disabled`、`disabled -> published` | 已停用题目重新启用前应确认题库可用 |
| `POST /api/admin/v1/service-tickets/{ticketId}/reply` | `service_ticket.ticket_status` | `waiting`、`processing`、`resolved` | 回复后保持 `processing` 或由 `resolved` 重开到 `processing` | `closed` 工单不允许直接回复，需先重开或新建 |
| `POST /api/admin/v1/reports/{reportId}/handle` | `risk_report_record.status` | `pending`、`processing` | `processing`、`resolved`、`rejected` | 普通处理建议 `pending -> processing -> resolved/rejected`；复审权限可允许 `resolved -> processing` |
