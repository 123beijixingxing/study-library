# 学习通后台联调执行看板模板

## 1. 文档说明

- 项目名称：学习通后台管理系统
- 对应前端工程：`study-library-admin`
- 文档目的：用于真实联调阶段按模块跟踪进度、责任人、阻塞项、冒烟结果与回归结论。

## 2. 推荐使用方式

建议每次联调启动前先做三件事：

1. 明确本轮联调模块
2. 填写当前环境、负责人、起止时间
3. 按模块更新“状态 / 阻塞 / 结论”字段

## 7. 冒烟路径模板

建议按以下路径做一轮完整冒烟：

1. 登录页 -> 登录成功
2. 仪表盘 -> 用户管理 -> 用户详情 -> 权限配置
3. 课程管理 -> 课程详情管理 -> 课程评价管理
4. 题库管理 -> 题目管理 -> 题目详情配置 -> 版本审计总台
5. 练习试卷 -> 试卷明细与分析 -> 方案保存 / 快照导出
6. 园地管理 -> 火热审核 / 分享审核 / 举报审核
7. 会员管理 -> 套餐 / 成长规则 / 学习记录
8. 消息管理 -> 模板管理
9. 系统管理 -> 统计页 -> 日志页

## 8. 联调完成判定标准

当满足以下条件时，可认为当前联调批次完成：

1. 登录链路通过
2. 至少 1 条主数据 CRUD 链路通过（用户 / 课程 / 题库 / 试卷）
3. 至少 1 条审核运营链路通过（园地 / 举报）
4. 统计或日志链路通过
5. `yarn type-check` 通过
6. `yarn build` 通过
