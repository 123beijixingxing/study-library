# 学习通状态流转测试用例清单

## 1. 文档说明

- 项目名称：学习通管理项目
- 文档版本：V1.1
- 编写日期：2026-04-24
- 对应文档：`docs/需求设计文档/学习通状态流转图说明.md`
- 对应文档：`docs/需求设计文档/学习通接口清单.md`
- 对应文档：`docs/需求设计文档/学习通前后端状态码规范.md`
- 文档目的：围绕关键业务状态机输出可执行测试清单，覆盖正常流转、非法流转、重复操作和异常分支，作为测试设计与联调验收的依据。

## 2. 测试设计原则

1. 每条状态机至少覆盖“创建初始状态”“正常流转”“非法流转”“重复操作”四类用例。
2. 状态流转测试需要同时验证数据库状态、接口响应、页面展示和日志记录四个层面。
3. 对高风险流程如审核、支付、消息发送、工单关闭、举报处理，必须增加越权和重复提交测试。
4. 对自动流转流程如验证码过期、会员到期、定时消息发送，必须覆盖定时任务或时间推进场景。

## 3. 测试用例字段说明

| 字段 | 说明 |
| --- | --- |
| 用例编号 | 建议按模块前缀命名，如 `COURSE-001` |
| 测试对象 | 被验证的状态实体 |
| 测试类型 | 正常流转、非法流转、重复操作、异常场景 |
| 前置条件 | 执行前准备的数据状态 |
| 操作步骤 | 可直接转为测试步骤 |
| 预期结果 | 包括接口、状态、前端表现、日志等 |
| 关联错误码 | 对应失败场景建议校验的错误码 |

## 6. 自动化测试建议

### 6.1 接口自动化优先项

- 认证：登录失败、验证码过期、重复发送验证码
- 课程：前台访问未上架课程、课程评价重复提交
- 练习：重复提交试卷、会员试卷权限拦截
- 客服：关闭工单继续回复、已解决工单重开
- 举报：非待审对象重复处理

### 6.2 数据校验建议

- 校验状态字段是否按预期变更，如 `pending -> approved`
- 校验冗余统计字段是否同步更新，如评价数、错题数、评论数
- 校验关键时间字段是否落库，如 `sendTime`、`handledAt`、`readTime`

### 6.3 日志校验建议

- 后台关键操作需要校验操作日志是否生成
- 工单处理、举报处理、课程上下架需要校验操作人和操作时间
- 消息发送失败需校验错误日志和 `requestId`

## 7. 联调验收建议

1. 联调阶段优先跑 P1 模块状态流转：课程、练习、会员、消息、客服、举报。
2. 对前台受限资源的访问，需同时确认接口返回码、页面提示和按钮显隐一致。
3. 对自动流转场景，建议提供测试开关或缩短时间阈值，便于验证过期和定时任务逻辑。
