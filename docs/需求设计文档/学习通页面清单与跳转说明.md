# 学习通页面清单与跳转说明

## 1. 文档说明

- 项目名称：学习通管理项目
- 文档版本：V1.1
- 编写日期：2026-04-23
- 对应需求文档：`docs/需求设计文档/学习通需求文档.md`
- 对应字段文档：`docs/需求设计文档/学习通页面字段说明.md`
- 对应接口文档：`docs/需求设计文档/学习通接口清单.md`
- 参考原型目录：`docs/UI交互设计`
- 文档目的：梳理前后台页面清单、原型映射关系、拓展页面规划、页面入口与页面跳转链路，支撑后续原型细化、前端路由设计与测试用例设计。

## 2. 页面规划原则

1. 以前台用户使用链路和后台管理操作链路为主线进行页面拆分。
2. 已存在原型页面优先沿用当前命名，避免后续文档与原型不一致。
3. 需求和拓展需求中已提及但原型中尚未单独体现的页面，统一纳入“待补充页面”管理。
4. 弹窗类功能如扫一扫、联系客服、练习弹窗，默认视为页面内弹层，不单独计入一级导航页面，除非后续产品另行拆页。
5. 优先补齐搜索结果、课程详情、练习中心、会员消息客服、后台运营治理等 P1 页面，确保主业务闭环可落地。

## 7. 典型用户流程

### 7.1 前台用户主流程

1. 游客进入登录页面。
2. 用户完成登录或先注册、找回密码后登录。
3. 登录成功后进入首页浏览推荐内容。
4. 用户通过搜索结果页、学习页或课程分类页找到目标课程。
5. 用户进入课程详情页查看章节、学习进度和课程评价。
6. 用户可继续学习，或进入练习中心完成章节练习和查看结果。
7. 用户通过我的页查看学习记录、会员权益、消息通知和客服会话。

### 7.2 前台用户成长与服务流程

1. 用户进入我的页并打开会员中心。
2. 用户查看当前会员等级、成长值、权益和续费入口。
3. 用户进入签到任务页完成签到或成长任务。
4. 用户在我的消息页查看系统通知，在在线客服页提交咨询或查看 FAQ。

### 7.3 后台管理员主流程

1. 管理员进入后台登录页并登录。
2. 登录后进入仪表盘查看整体运营数据。
3. 管理员根据职责进入用户、首页、课程、园地、会员、系统、题库与练习、运营与风控等模块。
4. 在模块中完成增删改查、审核、配置、组卷、消息发送、举报处理、统计或日志查看操作。

## 8. 页面与原型映射结论

- 当前原型已经覆盖前台 5 个一级页面和后台 10 个一级页面，适合用于展示主框架。
- 需求文档和拓展需求中已补充搜索结果、课程详情、练习中心、会员中心、我的消息、在线客服、题库管理、搜索运营、举报审核等二级页面，后续需继续补原型。
- 前台优先建议补充：搜索结果页、课程详情页、练习中心页、会员中心页、我的消息页、在线客服页。
- 后台优先建议补充：用户详情页、课程详情管理页、题库管理页、搜索运营配置页、举报审核页、消息管理页、客服管理页。
- 学习页顶部右侧按钮的功能命名需尽快统一，否则会影响原型、路由和测试用例编写。
