# 学习通后台模块级联调任务拆解清单

## 1. 文档说明

- 项目名称：学习通后台管理系统
- 对应前端工程：`study-library-admin`
- 文档目的：按后台模块拆解真实接口联调任务、验证点和交付标准，便于按阶段推进联调工作。

## 2. 联调分阶段建议

建议按以下顺序推进：

1. 登录鉴权与基础壳层
2. 主数据模块：用户、课程、题库、试卷
3. 审核运营模块：园地、举报、搜索运营、推荐位
4. 会员、消息、客服、系统配置
5. 分析与审计模块：统计、日志、版本审计、分析资产库

## 3. 模块级联调任务

### 3.1 登录与鉴权

接口范围：

- `POST /api/admin/v1/auth/login`
- `POST /api/admin/v1/auth/refresh`
- `POST /api/admin/v1/auth/logout`
- `GET /api/admin/v1/auth/me`

联调任务：

1. 登录接口接入真实 token / refreshToken
2. 刷新 token 接口验证 401 自动续签逻辑
3. 退出登录接口验证 session 清理
4. 用户信息接口验证头部用户名与角色显示

验收点：

- 登录成功后进入后台首页
- token 过期能自动刷新
- 刷新失败自动退回登录页

### 3.2 用户管理

接口范围：

- 用户列表、详情、编辑、状态切换、删除、权限配置

联调任务：

1. 用户列表分页与筛选条件联调
2. 新增/编辑用户表单字段回显与保存
3. 用户详情基础信息、学习记录、收藏统计回显
4. 角色权限数据结构适配

验收点：

- 列表查询与分页正确
- 详情页与列表数据一致
- 权限保存后重新进入页面仍正确

### 3.3 课程与章节管理

接口范围：

- 课程列表、课程详情、章节列表、章节保存、课程评价

联调任务：

1. 课程列表分页、状态、分类筛选联调
2. 课程详情回显讲师、简介、章节数据
3. 章节新增/修改/排序保存
4. 课程评价状态切换联调

验收点：

- 课程管理页与课程详情页字段一致
- 章节保存后列表即时回显

### 3.4 题库、题目与版本审计

接口范围：

- 题库列表
- 题目列表、题目详情、题目导入、题目状态切换
- 版本历史、版本复制、版本回滚、审计摘要导出

联调任务：

1. 题库列表与题目数量字段联调
2. 题目详情页的选项、答案、解析字段联调
3. 题目导入接口返回成功/失败条数联调
4. 版本历史接口返回版本组链路
5. 审计总台导出数据字段与真实接口比对

验收点：

- 题目详情保存后再次进入数据一致
- 版本链可以完整展示来源关系
- 回滚生成新版本而不是覆盖原版本

### 3.5 练习试卷与分析资产库

接口范围：

- 试卷列表、试卷详情、分析、错题分析、同课程试卷对比

联调任务：

1. 试卷列表筛选与保存联调
2. 试卷详情页的题目明细、组卷规则联调
3. 分析页成绩分布、趋势、错题热度、课程内排行联调
4. 分析快照导出数据结构联调
5. 快照回放和方案复用验证 payload 字段完整性

验收点：

- 试卷明细页可完整回显真实统计数据
- 导出快照内容与页面筛选一致

### 3.6 园地、审核与举报

接口范围：

- 园地内容列表、审核、举报列表、举报处理

联调任务：

1. 火热/分享内容列表字段联调
2. 审核状态变更联调
3. 举报处理结果、备注字段联调

验收点：

- 审核后列表状态即时刷新
- 举报处理记录可复查

### 3.7 首页、搜索运营、推荐位、敏感词

接口范围：

- 首页配置、搜索运营、推荐位配置、敏感词管理

联调任务：

1. 首页推荐模块和排序联调
2. 热搜词、推荐位、敏感词的新增/编辑/状态切换联调
3. 字段 adapter 校验

验收点：

- 保存后界面回显一致
- 状态切换即时更新

### 3.8 会员、消息、客服、系统

接口范围：

- 会员列表、套餐、成长规则、消息中心、消息模板、客服、系统配置

联调任务：

1. 会员列表与套餐字段联调
2. 成长规则触发类型和状态联调
3. 消息、模板、客服记录、系统配置联调

验收点：

- 列表和详情字段一致
- 重点开关配置保存后重新进入页面正确

### 3.9 统计与日志

接口范围：

- 统计概览、趋势、日志列表

联调任务：

1. 统计页总量、趋势、章节热度数据联调
2. 日志页分页、筛选和时间字段联调

验收点：

- 趋势数据可正常渲染
- 日志时间与排序正确

## 4. 每个模块统一联调步骤

建议每个模块都按以下固定流程执行：

1. 打开页面，先验证列表查询
2. 再验证新增/编辑/保存
3. 再验证状态切换或删除
4. 再验证详情页/分析页回显
5. 最后在浏览器 Network 中核对请求参数和返回字段

## 5. 联调问题记录建议

建议为每个模块至少记录：

- 接口地址
- 请求参数问题
- 返回字段问题
- 枚举值问题
- 时间字段问题
- 权限问题
- 已修复 / 待后端处理状态

## 6. 收尾建议

模块联调完成后，建议统一做三件事：

1. 跑一次 `yarn type-check`
2. 跑一次 `yarn build`
3. 对照 `docs/需求设计文档/学习通后台交付与验收清单.md` 做页面级验收
