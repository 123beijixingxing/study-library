# 学习通接口资产交付说明

## 1. 交付范围

当前接口资产已形成以下几类交付物：

- OpenAPI 草案：`docs/需求设计文档/学习通OpenAPI草案.yaml`
- Postman 基础集合：`docs/需求设计文档/学习通接口联调-PostmanCollection草案.json`
- Postman 扩展集合：`docs/需求设计文档/学习通接口联调-PostmanCollection扩展草案.json`
- Postman 环境：`docs/需求设计文档/学习通接口联调-Postman环境草案.json`
- Apifox 导入说明：`docs/需求设计文档/学习通Apifox导入说明.md`
- 成功响应示例：`docs/需求设计文档/学习通接口成功响应示例集.md`
- 错误响应示例：`docs/需求设计文档/学习通接口错误响应示例集.md`
- 接口联调报文：`docs/需求设计文档/学习通接口联调报文集.md`
- DTO 字典：`docs/需求设计文档/学习通接口DTO字段字典.md`
- 前后端样板：`docs/需求设计文档/代码模板示例/`

## 2. 推荐使用顺序

1. 先阅读 `docs/需求设计文档/学习通接口清单.md` 了解全量接口范围。
2. 再查看 `docs/需求设计文档/学习通OpenAPI草案.yaml` 对接路径、参数和 schema。
3. 前端/后端研发分别参考 `docs/需求设计文档/代码模板示例/` 中的类型与骨架。
4. 联调阶段优先导入 `docs/需求设计文档/学习通接口联调-PostmanCollection扩展草案.json`。
5. 错误处理统一参考 `docs/需求设计文档/学习通前后端状态码规范.md` 和 `docs/需求设计文档/学习通接口错误响应示例集.md`。

- OpenAPI：已覆盖前后台核心路径、常见请求体、主要分页/详情/状态响应 schema

- 联调材料：已具备基础集合、扩展集合、环境变量、响应示例与报文示例
