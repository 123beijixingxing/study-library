# 操作手册

## 1. 面向角色

- 产品：查看接口范围、状态规则、页面字段映射
- 前端：导入 Postman/Apifox，参考 TS 类型和 API 骨架
- 后端：参考 OpenAPI、DTO 字典、状态码规范和 Java 骨架
- 测试：导入 CSV 用例，参考联调报文和错误响应示例

## 2. 推荐使用顺序

### 2.1 前后端研发

1. 阅读 `docs/需求设计文档/学习通OpenAPI草案.yaml`
2. 阅读 `docs/需求设计文档/学习通接口DTO字段字典.md`
3. 阅读 `docs/需求设计文档/学习通前后端状态码规范.md`
4. 使用 `docs/需求设计文档/代码模板示例/` 启动工程骨架
5. 使用 `docs/需求设计文档/学习通接口联调-PostmanCollection扩展草案.json` 开始联调

### 2.2 测试

1. 阅读 `docs/需求设计文档/学习通状态流转图说明.md`
2. 导入 `docs/需求设计文档/学习通状态流转测试用例清单-P1冒烟.csv`
3. 联调稳定后导入 `docs/需求设计文档/学习通状态流转测试用例清单-完整回归.csv`
4. 结合 `docs/需求设计文档/学习通接口错误响应示例集.md` 校验异常场景

## 4. 常见使用场景

### 4.1 只做接口阅读

- 直接阅读 `docs/需求设计文档/学习通接口清单.md`
- 如需结构化定义，再看 `docs/需求设计文档/学习通OpenAPI草案.yaml`

### 4.2 只做前台联调

- 导入 Postman 扩展集合后，只使用 `App` 相关分组
- 使用 `appToken` 环境变量

### 4.3 只做后台联调

- 导入 Postman 扩展集合后，只使用 `Admin` 相关分组
- 使用 `adminToken` 环境变量

## 5. 注意事项

1. 交付包中的代码模板为示例骨架，不是生产级工程。
2. OpenAPI 草案虽然已高度覆盖，但仍属于需求设计阶段资产，正式开发时可继续细化 example、security、error response。
3. 所有联调响应结构应优先以状态码规范和响应示例集为准。
